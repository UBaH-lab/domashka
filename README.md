## Инструкция по установке и запуску проекта

### 1. Клонирование репозитория (если есть)
Если проект хранится на GitHub или другом репозитории:

```bash
git clone <URL_репозитория>
cd <имя_проекта>
Если проект уже есть локально — просто перейдите в папку проекта.

2. Установка Python и необходимых инструментов
Убедитесь, что у вас установлена версия Python >= 3.13.
Убедитесь, что установлен менеджер пакетов pip.
3. Установка зависимостей
Через Poetry (если используется pyproject.toml)
Установите Poetry (если ещё не установлен):
bash

pip install poetry
Или следуйте официальной инструкции: https://python-poetry.org/docs/#installation

Установите зависимости и виртуальное окружение:
bash

poetry install
Активируйте виртуальную среду:
bash

poetry shell
Или запускайте команды через poetry run.

Через requirements.txt (если есть)
Установите зависимости командой:
bash

pip install -r requirements.txt
4. Запуск проекта
Обычно проект запускается командой:
bash

python main.py
Или через Poetry:
bash

poetry run python main.py
Если в документации указаны другие команды — используйте их.

##Модули и их описание
1. Модуль processing
filter_by_state
Принимает список словарей и опциональное значение для ключа state.
Возвращает новый список словарей, где state совпадает с заданным.
2. Модуль masks
get_mask_card_number
Принимает номер карты, возвращает маскированный номер, показывая только первые 6 и последние 4 цифры.
get_mask_account
Принимает номер счёта, возвращает маскированный вид — показываются только последние 4 цифры, перед ними две звёздочки.
3. Модуль widget
mask_account_card
Принимает строку с типом (карта или счёт) и номером.
Возвращает строку с правильной маской в зависимости от типа (карта или счёт).

Обработка отсутствующих данных.
get_mask_account:
Маскирование номеров счёта разных форматов и длин, обработка коротких номеров.
Модуль widget

mask_account_card:
Правильное определение типа и маскировка.
Обработка некорректных или ошибочных входных данных.
Модуль get_date

Проверка преобразования форматов дат:
Разные форматы дат.
Граничные случаи.
Обработка отсутствующих дат.

2. Модуль processing

filter_by_state:
Проверка фильтрации по статусу с различными сценариями.
sort_by_date:
Сортировка по дате в разных направлениях.
Обработка одинаковых дат и некорректных данных.

3. Модуль widget 

В этом файле тестируются две функции: mask_account_card и get_date.

mask_account_card
Проверяет, правильно ли функция маскирует номера счетов и карт. Тесты покрывают различные сценарии:

Полностью замаскированные номера карт (например, последние 4 цифры оставляются открытыми, а остальное закрыто звездочками).
Номера счетов, где маскируются только последние 4 цифры.
Случаи, когда строка не содержит номера или не поддается маскированию (например, "Нет данных" или пустая строка).
Проверяет, что при некорректных или недопустимых данных функция возвращает ожидаемый результат.
get_date
Проверяет правильность обработки и преобразования дат:

Преобразует даты из формата ISO 8601 (например, "2024-03-11T02:26:18.671407") в формат "день.месяц.год" (например, "11.03.2024").
Обрабатывает даты без времени.
Проверяет, что при некорректных датах или ошибочных строках функция выбрасывает исключение ValueError.
Эти тесты помогают убедиться, что функции работают корректно в различных ситуациях и корректно обрабатывают неожиданные или ошибочные входные данные.
