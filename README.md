## Описание проекта

Данный проект реализует функции, необходимые для личного кабинета банковского клиента:
- Маскировка номеров карт и счетов
- Фильтрация данных по статусу
- Сортировка по датам
- Валидация и преобразование дат

В проекте реализованы модули:
- `processing`
- `masks`
- `widget`

Также включены автоматические тесты для проверки корректности работы функций.

## Установка

### 1. Клонирование репозитория

Если проект хранится в репозитории, выполните:

```bash
git clone <URL_репозитория>
cd <имя_проекта>
Если проект уже есть у вас локально — перейдите в папку проекта.

2. Установка зависимостей
Через Poetry (рекомендуется)
Установите Poetry, если ещё не установлен:
bash

Или следуйте официальной инструкции по установке Poetry.

bash

Активируйте виртуальную среду:
bash

или запускайте проект через:

bash

Если в проекте есть файл requirements.txt, выполните:

bash

pip install -r requirements.txt
Запуск
Для запуска проекта используйте команду:

bash

или с помощью Poetry:

bash

poetry run python main.py
Обратите внимание, что конкретные команды могут зависеть от настроек вашего проекта.

Тестирование
Как запустить тесты
Убедитесь, что установлен pytest:
bash

pip install pytest
Запустите тесты:
bash

pytest test_module.py
Что проверяют тесты
Модуль masks

get_mask_card_number: правильность маскировки номеров карт разных форматов, обработка некорректных данных.
get_mask_account: корректность маскировки номеров счетов разных длины и форматов.
Модуль widget

mask_account_card: правильная маскировка по типу (карта или счет), обработка ошибки и некорректных данных.
Общие функции

filter_by_state: фильтрация данных.
sort_by_date: сортировка по дате.
get_date: преобразование даты из ISO в формат "день.месяц.год".
Замечания
Все функции протестированы и проходят автоматические проверки.
В случае ошибок или некорректных данных функции должны возвращать логичные значения или бросать исключения (по описанию).
